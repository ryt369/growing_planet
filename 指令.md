# 生长星球项目指令汇总

## 一、项目基础结构

我要做一个纯前端交互项目，用 HTML + CSS + JavaScript（三件套，不依赖任何第三方库）。

项目名称：生长星球

功能概述：

1. 页面中央显示一个圆形星球，用 SVG 绘制。
2. 点击星球任意位置，会长出植物（小树、小草、花）。
3. 拖拽星球可以旋转视角。
4. 滚轮可以缩放。
5. 所有植物数据存入 localStorage，刷新不丢失。

生成项目的基础 HTML 结构，包含：

- 一个用于显示星球的 container。
- CSS 基础样式（深色星空背景，星球居中对齐）。
- JS 文件的基础框架（注释写好各个功能分区）。

要求：简洁清晰，便于后续扩展。

## 二、绘制星球 SVG

绘制星球的 SVG，要求：

1. 圆形，直径 500px，带径向渐变（从 `#4a7c59` 到 `#2d4a3e`，看起来像有生命的星球）。
2. 外部加一层淡淡的发光光晕（使用 `box-shadow` 或 `filter: drop-shadow`）。
3. 星球表面增加一些随机的小点或纹理，让星球不单调，但不要喧宾夺主。

SVG 直接写在 HTML 里，方便后续用 JavaScript 操作。

## 三、点击星球种植植物

实现点击星球种植物功能，需求：

1. 点击星球时，获取点击位置相对于星球中心的坐标，并归一化到 `-1` 到 `1`。
2. 判断点击是否在球面内（距离中心 `< 1`）。
3. 在点击位置生成一个植物，植物用 emoji 表示，从 `🌱`、`🌿`、`🌲`、`🌸` 中随机选择一个。
4. 植物大小随机，为 `0.3` 到 `0.8` 倍。
5. 每个植物要有一个唯一 ID，并存储在数组里。

植物数据结构：

```js
{
  id: 0,
  x: 0.3,        // 归一化坐标
  y: -0.5,
  emoji: '🌱',
  size: 0.5,
  rotation: 45   // 植物自身旋转角度
}
```

需要实现：

- 点击事件监听。
- 坐标计算逻辑。
- 植物生成函数。
- 植物渲染函数（把植物显示在星球上）。

## 四、星球拖拽旋转

实现星球拖拽旋转功能，要求：

1. 鼠标按下星球时开始拖拽。
2. 鼠标移动时星球跟随旋转，水平拖拽控制旋转角度。
3. 鼠标松开时停止拖拽。
4. 旋转时，所有植物也跟着旋转，它们相对于星球的位置不变。

技术方案：

- 监听 `mousedown`、`mousemove`、`mouseup`。
- 使用一个 `state.rotation` 变量存储当前旋转角度（度）。
- 每次旋转时，使用 CSS `transform: rotate()` 旋转整个星球容器。
- 植物位置使用旋转矩阵计算：

```js
newX = x * cos(θ) - y * sin(θ);
newY = x * sin(θ) + y * cos(θ);
```

## 五、滚轮缩放

加入滚轮缩放功能，要求：

1. 滚轮向上滚动，星球放大，最大为 2 倍。
2. 滚轮向下滚动，星球缩小，最小为 0.5 倍。
3. 缩放中心为星球中心。
4. 缩放到极限值时停止，防止溢出。

使用 CSS `transform: scale()` 实现，并注意与旋转变换叠加。

## 六、localStorage 数据存储

加入 localStorage 存储功能，要求：

1. 每次植物数组变化时，自动保存到 localStorage。
2. 页面加载时，从 localStorage 读取数据并恢复现场。
3. 存储 key 使用 `planet-data`。
4. 存储内容包括植物数组、当前旋转角度和缩放倍数。

数据结构示例：

```js
{
  plants: [...],
  rotation: 45,
  zoom: 1.2
}
```

注意使用 JSON 进行序列化和反序列化。

## 七、空闲检测和夜晚模式

实现空闲检测和夜晚模式，需求：

1. 用户停止操作（无点击、无拖拽、无滚动）10 秒后，自动进入夜晚模式。
2. 夜晚模式下，背景渐变到深蓝或黑色，星球亮度降低，植物使用 `box-shadow` 或 `filter: drop-shadow` 发出柔和光晕。
3. 用户点击或操作后，立即回到白天模式。
4. 白天到夜晚的过渡使用 CSS `transition`，实现平滑变化。

## 八、流星雨特效

加入流星雨特效。

触发方式：双击星球任意位置。

特效效果：

1. 从屏幕右上角随机位置生成 15～20 颗流星。
2. 每颗流星由一条斜向下的发光线条和一个头部光点组成。
3. 流星速度和长度随机。
4. 使用 Canvas 绘制，并与粒子系统共用 Canvas。
5. 持续 3 秒后自动结束。

要求：

- 流星轨迹为直线，角度在 30°～60°之间。
- 流星尾部使用透明度渐变逐渐消失。
- 不干扰已有的粒子特效。
